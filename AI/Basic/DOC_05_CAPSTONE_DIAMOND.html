What You're Building

A document AI assistant — upload files, ask questions, get cited answers

The capstone is a production-ready RAG chatbot. Users upload any documents (PDF, TXT, Markdown), the system indexes them, and they can ask natural language questions and receive streamed answers with source citations — all from a clean Streamlit interface backed by a FastAPI server.

🎯 The Product

Document AI Assistant: Upload your files → system chunks, embeds, and indexes them into LanceDB → you ask questions in a chat interface → LangChain retrieves the top-5 most relevant chunks → feeds them to Llama 3.2 via Ollama → streams the answer back token by token with source citations → fully Dockerized, runs on any machine.

Phase 1 — Vector Store Layer

LanceDB wrapper: chunk, embed, store, search
python · backend/vectorstore.py
import lancedb, uuid
from pathlib     import Path
from typing      import List, Dict
from sentence_transformers import SentenceTransformer

DB_PATH    = "./data/vectordb"
EMBED_MODEL= "all-MiniLM-L6-v2"   # free, fast, good enough

class VectorStore:
  def __init__(self):
    self.db  = lancedb.connect(DB_PATH)
    self.emb = SentenceTransformer(EMBED_MODEL)
    # Open or create the chunks table
    if "chunks" in self.db.table_names():
      self.table = self.db.open_table("chunks")
    else:
      self.table = None

  def add_chunks(self, chunks: List[str], source: str) -> int:
    """Embed and store a list of text chunks."""
    records = [{
      "id":     str(uuid.uuid4()),
      "text":   c,
      "source": source,
      "vector": self.emb.encode(c).tolist()
    } for c in chunks]

    if self.table is None:
      self.table = self.db.create_table("chunks", data=records)
    else:
      self.table.add(records)
    return len(records)

  def search(self, query: str, top_k: int = 5) -> List[Dict]:
    """Return top-K most similar chunks with source info."""
    if self.table is None:
      return []
    qvec = self.emb.encode(query).tolist()
    rows = self.table.search(qvec).limit(top_k).to_pandas()
    return rows[["text", "source", "_distance"]].to_dict("records")

  def clear(self):
    """Drop and recreate the table."""
    if "chunks" in self.db.table_names():
      self.db.drop_table("chunks")
    self.table = None

Phase 2 — RAG Pipeline

LangChain: ingest documents, generate streaming answers
📐 Diagram 2 — RAG Data Flow: From Question to Streamed Answer
python · backend/rag.py
from typing      import Generator, List
from langchain_openai   import ChatOpenAI
from langchain.prompts  import ChatPromptTemplate
from langchain.schema.output_parser import StrOutputParser
from langchain_text_splitters import RecursiveCharacterTextSplitter
from vectorstore import VectorStore

# ── LLM: Ollama via OpenAI-compatible API ────────────────────
# To switch to OpenAI: remove base_url, set real api_key
LLM = ChatOpenAI(
  base_url = "http://ollama:11434/v1",  # docker-compose hostname
  model    = "llama3.2",
  api_key  = "ollama"
)

SPLITTER = RecursiveCharacterTextSplitter(
  chunk_size=512, chunk_overlap=50
)

RAG_PROMPT = ChatPromptTemplate.from_template("""
You are a helpful assistant. Answer ONLY using the context below.
If the answer isn't in the context, say "I couldn't find that in the documents."
Always end with: 📎 Source: [list the source files you used]

Context:
{context}

Conversation history:
{history}

Question: {question}
Answer:""")

VS = VectorStore()  # singleton — created once at import time

def ingest(text: str, source: str) -> int:
  """Split text into chunks and store in LanceDB."""
  chunks = SPLITTER.split_text(text)
  return VS.add_chunks(chunks, source)

def chat_stream(question: str, history: List[dict]) -> Generator:
  """Retrieve context, build prompt, stream tokens."""
  # 1. Retrieve
  hits    = VS.search(question, top_k=5)
  context = "\n\n".join(h["text"] for h in hits)
  sources = list({h["source"] for h in hits})

  # 2. Format last 3 turns of history
  hist_text = "\n".join(
    f"Human: {h['human']}\nAssistant: {h['assistant']}"
    for h in history[-3:]
  )

  # 3. Stream answer
  chain = RAG_PROMPT | LLM | StrOutputParser()
  for chunk in chain.stream({
    "context": context, "history": hist_text, "question": question
  }):
    yield chunk

  # 4. Append source sentinel (frontend parses this out)
  yield f"\n\n__SOURCES__:{','.join(sources)}"

Phase 3 — FastAPI Backend

Three endpoints: upload, chat (streaming), health
python · backend/main.py
import io, os
from fastapi          import FastAPI, UploadFile, File, HTTPException
from fastapi.responses import StreamingResponse
from pydantic          import BaseModel
from typing            import List
import PyPDF2
from rag import ingest, chat_stream

app = FastAPI(title="RAG Chatbot API", version="1.0")

# ── Health ────────────────────────────────────────────────────
@app.get("/health")
def health():
  return {"status": "ok", "model": "llama3.2", "backend": "ollama"}

# ── Upload & Index ────────────────────────────────────────────
@app.post("/upload")
async def upload(file: UploadFile = File(...)):
  try:
    content = await file.read()
    if file.filename.endswith(".pdf"):
      reader = PyPDF2.PdfReader(io.BytesIO(content))
      text = "\n".join(p.extract_text() or "" for p in reader.pages)
    else:
      text = content.decode("utf-8", errors="ignore")

    n = ingest(text, file.filename)
    return {"status": "ok", "file": file.filename, "chunks": n}
  except Exception as e:
    raise HTTPException(status_code=500, detail=str(e))

# ── Streaming Chat ────────────────────────────────────────────
class ChatRequest(BaseModel):
  question: str
  history:  List[dict] = []

@app.post("/chat")
async def chat(req: ChatRequest):
  def gen():
    for token in chat_stream(req.question, req.history):
      yield token
  return StreamingResponse(gen(), media_type="text/plain")
⚡ Gotcha — Ollama hostname in Docker

When running inside docker-compose, the Ollama service is reachable at http://ollama:11434/v1, not localhost. For local dev (no Docker), change it to http://localhost:11434/v1. Use an env variable to switch between the two.

Phase 4 — Streamlit Frontend

Upload sidebar + streaming chat + source citations
python · frontend/app.py
import streamlit as st, requests, os

API = os.getenv("API_URL", "http://localhost:8000")

st.set_page_config(
  page_title="📚 Doc AI Assistant",
  page_icon="📚",
  layout="wide"
)

# ── Sidebar: upload ───────────────────────────────────────────
with st.sidebar():
  st.header("📂 Upload Documents")
  files = st.file_uploader(
    "PDF or TXT files",
    type=["pdf","txt","md"],
    accept_multiple_files=True
  )
  if files and st.button("📥 Index Documents", type="primary"):
    for f in files:
      with st.spinner(f"Indexing {f.name}..."):
        r = requests.post(
          f"{API}/upload",
          files={"file": (f.name, f.getvalue())}
        )
      if r.status_code == 200:
        d = r.json()
        st.success(f"✅ {d['file']}: {d['chunks']} chunks")
      else:
        st.error(f"❌ Failed: {f.name}")

  st.divider()
  # Health indicator
  try:
    h = requests.get(f"{API}/health", timeout=2).json()
    st.success(f"🟢 {h['model']} via {h['backend']}")
  except:
    st.error("🔴 Backend offline")
  st.divider()
  if st.button("🗑️ Clear Chat"):
    st.session_state.messages = []; st.rerun()

# ── Main: chat ────────────────────────────────────────────────
st.title("📚 Document AI Assistant")
st.caption("Upload documents on the left, then ask questions here.")

if "messages" not in st.session_state:
  st.session_state.messages = []

for m in st.session_state.messages:
  with st.chat_message(m["role"]):
    st.markdown(m["content"])
    if m.get("sources"):
      with st.expander("📎 Sources", expanded=False):
        for s in m["sources"]:
          st.caption(f"• {s}")

if question := st.chat_input("Ask a question about your documents..."):
  # Build history list for context
  hist = [
    {"human": st.session_state.messages[i]["content"],
     "assistant": st.session_state.messages[i+1]["content"]}
    for i in range(0, len(st.session_state.messages)-1, 2)
  ]
  st.session_state.messages.append({"role":"user","content":question})
  with st.chat_message("user"): st.markdown(question)

  with st.chat_message("assistant"):
    box     = st.empty()
    partial = ""
    sources = []
    with requests.post(f"{API}/chat",
        json={"question": question, "history": hist},
        stream=True, timeout=120) as r:
      for chunk in r.iter_content(chunk_size=None, decode_unicode=True):
        if "__SOURCES__:" in chunk:
          sources = chunk.replace("__SOURCES__:","").strip().split(",")
        else:
          partial += chunk
          box.markdown(partial + "▌")
    box.markdown(partial)
    if sources:
      with st.expander("📎 Sources", expanded=True):
        for s in sources: st.caption(f"• {s}")

  st.session_state.messages.append({
    "role":"assistant","content":partial,"sources":sources})

Phase 6 — Docker Deployment

One command to run everything, anywhere
yaml · docker-compose.yml
version: '3.8'

services:

  # Ollama — local LLM server
  ollama:
    image: ollama/ollama
    ports:
      - "11434:11434"
    volumes:
      - ollama_data:/root/.ollama
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:11434"]
      interval: 10s
      retries: 5

  # FastAPI backend
  backend:
    build: ./backend
    ports:
      - "8000:8000"
    volumes:
      - ./data:/app/data   # persist LanceDB across restarts
    depends_on:
      ollama:
        condition: service_healthy
    environment:
      - OLLAMA_HOST=http://ollama:11434/v1
    command: uvicorn main:app --host 0.0.0.0 --port 8000

  # Streamlit frontend
  frontend:
    build: ./frontend
    ports:
      - "8501:8501"
    environment:
      - API_URL=http://backend:8000
    depends_on:
      - backend
    command: streamlit run app.py --server.address 0.0.0.0

volumes:
  ollama_data: {}

Bonus Challenges

Push further — these are the skills that separate good from great
⭐⭐ Medium
Swap LanceDB for Pinecone
Replace the VectorStore class backend with Pinecone. The LangChain pipeline and FastAPI endpoints stay identical. Only vectorstore.py changes. This proves the architecture is backend-agnostic.
⭐⭐ Medium
Add User Authentication
Add FastAPI OAuth2 with JWT tokens. Each user gets their own LanceDB table (namespace isolation). Documents uploaded by User A are invisible to User B. Add login/logout to the Streamlit sidebar.
⭐⭐⭐ Hard
Add a LangGraph Self-Correction Loop
Wrap the RAG pipeline in a LangGraph loop. After generating an answer, score it 1–10. If below 6, retrieve more chunks (top_k=8) and regenerate. Cap at 3 attempts. Log the improvement across attempts.
⭐⭐⭐ Hard
Migrate Frontend to Next.js
Rebuild the Streamlit UI in Next.js with the Vercel AI SDK. Keep the FastAPI backend unchanged. Add streaming with useChat, persist history to Supabase, and deploy to Vercel. Show the architecture still holds when the UI layer swaps.
🏆 Extension
Add Voice Input with Whisper
Add a voice recording component. Transcribe with Whisper (Hugging Face pipeline, local). Feed the transcript to the RAG chat. The app becomes voice-first — users ask questions by speaking, not typing.
🏆 Extension
Deploy to a Public Cloud
Deploy to Railway.app or Fly.io using the docker-compose.yml. Replace Ollama with OpenAI (one line) for cloud deployment since Ollama needs a GPU server. Get a public URL and share your working app.
