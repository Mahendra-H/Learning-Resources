The Big Picture

Read this before anything else — it changes how the rest sticks

Before writing a single line of code, understand this: these 4 tools are not competitors — they are layers. Each one solves a different problem at a different level of your app. Most confusion in this module comes from treating them as alternatives when they're actually complements.

🧠 Brain Anchor

LangChain

The assembly line for LLM logic

Without a framework, every AI feature you build is a tangled mess of manual prompt strings, API calls, and custom parsing scattered everywhere. LangChain gives you composable, swappable building blocks — and one operator that ties them all together.

The LCEL Pipe — Most Important Concept in LangChain

LCEL stands for LangChain Expression Language. The | pipe operator chains steps together. Each step's output becomes the next step's input — exactly like Unix shell pipes.

📐 Diagram 2 — The LCEL Pipeline: What Flows Between Steps
⚡ Gotcha — The | Is Not Bitwise OR

LangChain overrides Python's __or__ dunder method on all its objects. So prompt | llm returns a runnable chain, not a number. That's why it works — and why it looks like magic.

The 6 Building Blocks

Master these 6 concepts and you know 90% of LangChain.

Quick Start — Your First Chain

python · install: pip install langchain langchain-openai
from langchain_openai import ChatOpenAI
from langchain.prompts import ChatPromptTemplate
from langchain.schema.output_parser import StrOutputParser

# 1. Define each block separately
llm    = ChatOpenAI(model="gpt-4o-mini", temperature=0.7)
prompt = ChatPromptTemplate.from_template(
    "Explain {topic} to a {audience} in {style} style."
)
parser = StrOutputParser()

# 2. Snap them together with |
chain = prompt | llm | parser

# 3. Run it
result = chain.invoke({
    "topic":    "vector databases",
    "audience": "a 10-year-old",
    "style":    "storytelling"
})
print(result)  # Plain string — ready to display or pass to next chain
🏢 In the Wild

ChatGPT's Code Interpreter is an LLM + a Python execution tool, chained together. Perplexity is an LLM + a web search tool + a citation parser — all wired as a chain. Every AI feature that "does something" beyond text generation is using this tool-calling pattern.

LangGraph

When straight lines aren't enough

LangChain chains go forward: A → B → C → Done. That's perfect for 80% of tasks. But real agents need to loop back, branch left or right, or stop early based on a result. That's where chains break — and LangGraph begins.

📐 Diagram 4 — Linear Chain vs Graph Agent
🧠 Brain Anchor

LangChain = hiring form you fill out once and submit. LangGraph = the actual hiring process — screen → interview → reject-or-offer → negotiate → loop back if offer rejected. Real processes have cycles. LangGraph handles them.

The Research Agent — LangGraph State Machine

Here's a real LangGraph agent that generates research questions, answers them, checks quality, and loops until satisfied:

📐 Diagram 5 — LangGraph Research Agent State Machine
python · LangGraph state machine
from typing import TypedDict, List
from langgraph.graph import StateGraph, END

# 1. Define shared state — the "whiteboard" all nodes can read & write
class ResearchState(TypedDict):
    topic:      str
    questions:  List[str]
    answers:    List[str]
    draft:      str
    attempts:   int
    is_good:    bool

# 2. Define nodes — each reads state, returns updated state
def write_draft(state: ResearchState) -> ResearchState:
    qa = "\n".join([f"Q: {q}\nA: {a}" for q, a in zip(state["questions"], state["answers"])])
    draft = llm.invoke(f"Write a cohesive report:\n{qa}").content
    return {**state, "draft": draft, "attempts": state["attempts"] + 1}

def quality_check(state: ResearchState) -> ResearchState:
    result = llm.invoke(f"Is this high quality? YES or NO only:\n{state['draft']}").content
    return {**state, "is_good": "YES" in result.upper()}

# 3. Routing function — returns name of next node
def should_revise(state: ResearchState) -> str:
    if state["is_good"] or state["attempts"] >= 3:
        return "done"
    return "revise"

# 4. Build the graph
graph = StateGraph(ResearchState)
graph.add_node("draft",   write_draft)
graph.add_node("check",   quality_check)
graph.set_entry_point("draft")
graph.add_edge("draft", "check")
graph.add_conditional_edges("check", should_revise, {"done": END, "revise": "draft"})

app = graph.compile()
result = app.invoke({"topic": "climate change", "questions": [], "answers": [],
                     "draft": "", "attempts": 0, "is_good": False})
🏢 In the Wild

Devin (the AI software engineer), AutoGPT, and most "AI agent" products are fundamentally LangGraph-style loops: Think → Plan → Act → Observe → (satisfied? stop : loop). The "Think" phase is the LLM; the loop is LangGraph.

LlamaIndex

Give your AI access to your own data

LLMs know about the world up to their training cutoff. They know nothing about your company's policies, your codebase, or your notes. LlamaIndex bridges that gap — it connects LLMs to any private data source so they can reason over it.

🧠 Brain Anchor

Imagine hiring a brilliant consultant who's read every book ever written — but has never seen your company's internal docs. LlamaIndex is what hands them the internal docs before the meeting. The consultant (LLM) is still brilliant — now they're also relevant.

Module 1 Capstone Build

Combine all 4 tools into one working system

Don't just read. Build. This capstone uses LangGraph as the controller, LlamaIndex for retrieval, LangChain for generation, and Hugging Face for embeddings. It's the complete Module 1 brain in one system.

🎯 What You're Building

A Self-Improving Q&A System: User asks a question → LlamaIndex retrieves relevant docs → LangChain generates an answer → LangGraph checks quality → if not good enough, rephrases and retries (max 3 attempts) → returns answer with sources cited.

python · capstone_qa.py
from typing import TypedDict, List
from langgraph.graph import StateGraph, END
from llama_index.core import VectorStoreIndex, SimpleDirectoryReader
from langchain_openai import ChatOpenAI
from langchain.prompts import ChatPromptTemplate
from langchain.schema.output_parser import StrOutputParser

# ── Setup ─────────────────────────────────────────────
docs    = SimpleDirectoryReader("./my_docs").load_data()
index   = VectorStoreIndex.from_documents(docs)
engine  = index.as_query_engine(similarity_top_k=3)
llm     = ChatOpenAI(model="gpt-4o-mini")
parser  = StrOutputParser()

# ── State ─────────────────────────────────────────────
class QAState(TypedDict):
    question: str;  context: str;  sources: List[str]
    answer:   str;  quality: int;  attempts: int

# ── Nodes ─────────────────────────────────────────────
def retrieve(s):
    r = engine.query(s["question"])
    srcs = [n.metadata.get("file_name", "?") for n in r.source_nodes]
    return {**s, "context": r.response, "sources": list(set(srcs))}

def generate(s):
    chain = ChatPromptTemplate.from_template(
        "Context:\n{ctx}\n\nAnswer this question using the context: {q}"
    ) | llm | parser
    ans = chain.invoke({"ctx": s["context"], "q": s["question"]})
    return {**s, "answer": ans, "attempts": s["attempts"]+1}

def score(s):
    chain = ChatPromptTemplate.from_template(
        "Rate 1-10 (number only). 10=complete answer. 1=no answer.\n{a}"
    ) | llm | parser
    q = chain.invoke({"a": s["answer"]})
    return {**s, "quality": int("".join(filter(str.isdigit, q)) or "5")}

# ── Graph ─────────────────────────────────────────────
g = StateGraph(QAState)
g.add_node("retrieve", retrieve)
g.add_node("generate", generate)
g.add_node("score",    score)
g.set_entry_point("retrieve")
g.add_edge("retrieve", "generate")
g.add_edge("generate", "score")
g.add_conditional_edges("score",
    lambda s: "done" if s["quality"]>=6 or s["attempts"]>=3 else "retry",
    {"done": END, "retry": "retrieve"})

app = g.compile()
out = app.invoke({"question": "What is the refund policy?",
                  "context": "", "sources": [], "answer": "", "quality": 0, "attempts": 0})
print(out["answer"])
print(f"Sources: {', '.join(out['sources'])}")
print(f"Took {out['attempts']} attempt(s), quality score: {out['quality']}/10")
