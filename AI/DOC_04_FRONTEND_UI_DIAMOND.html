Gradio

Turn any Python function into a shareable web app in 5 minutes
🎬 Origin Story — Why It Was Built

Abubakar Abid was a PhD student at Stanford in 2019, struggling to share his machine learning research with collaborators who weren't engineers. Writing a Flask app for every demo took a full day. He wanted something that let him go from "working Python function" to "shareable web interface" in minutes. Gradio was born: wrap a function, get a UI. One decorator, one URL to share. Hugging Face acquired Gradio in 2021, integrating it tightly with the model hub — now every model on Hugging Face Spaces runs a Gradio interface. Today Gradio powers millions of AI demos globally and is the default "show your work" tool for AI researchers and engineers alike.

The Chatbot Pattern — Most Common Real Use

python · streaming chatbot with RAG — full example
import gradio as gr
from langchain_openai import ChatOpenAI
from langchain.memory import ConversationBufferWindowMemory
import requests

llm = ChatOpenAI(model="gpt-4o-mini", streaming=True)
API = "http://localhost:8000"  # your FastAPI from Module 2

def chat(message: str, history: list) -> str:
  # Call your Module 2 FastAPI backend
  resp = requests.post(f"{API}/ask",
    json={"question": message, "history": history},
    stream=True)
  # Stream tokens back to Gradio
  partial = ""
  for chunk in resp.iter_content(chunk_size=None, decode_unicode=True):
    partial += chunk
    yield partial  # Gradio renders each yield instantly

with gr.Blocks(title="AI Assistant", theme=gr.themes.Soft()) as demo:
  gr.Markdown("# 🤖 AI Assistant\nPowered by your Module 2 + 3 stack")
  with gr.Row():
    with gr.Column(scale=3):
      chatbot = gr.ChatInterface(
        fn=chat, type="messages",
        examples=["What is RAG?", "Explain LangChain in one sentence"],
        title=""
      )
    with gr.Column(scale=1):
      gr.Markdown("### ⚙️ Settings")
      temp  = gr.Slider(0, 1, value=0.7, label="Temperature")
      style = gr.Dropdown(["concise","detailed","ELI5"], label="Answer Style", value="concise")

demo.launch(share=True)

🔴 The Problem

A research team built a fine-tuned summarization model. Their grant reviewers wanted to evaluate it interactively — but the reviewers were domain experts, not engineers. Asking them to clone a repo and run Python was not an option. Demo day was in 48 hours.

🟢 The Gradio Solution

30 minutes to build a Gradio interface: a text area for input, a slider for summary length, a dropdown for academic/general/bullet format. launch(share=True) generated a public URL. Reviewers tested the model from their browsers for 3 hours with zero technical setup.

Result: Grant renewed. Total development time for the interface: 30 minutes. If they'd built a custom web app: 2+ days. Gradio was the only tool that could close the gap between "works in Jupyter" and "accessible to reviewers" in the available time.

Anti-Patterns — The Mistakes Everyone Makes

Anti-Pattern #1
🔴 Symptom

Shared the Gradio link but the demo is down the next morning. Colleague can't access it.

🟡 Root Cause

share=True links expire after 72 hours and require your local machine to stay running. The moment you close the laptop, the demo dies.

🟢 Fix

For persistent demos: deploy to Hugging Face Spaces (free, permanent). Or run launch(server_name="0.0.0.0") on a cloud VM with a static IP.

Anti-Pattern #2
🔴 Symptom

Response appears all at once after 10 seconds instead of streaming token by token. UX feels broken.

🟡 Root Cause

Function returns a string instead of yield-ing chunks. Gradio only streams when the function is a Python generator.

🟢 Fix

Use yield partial_response inside your function instead of building the full string and returning it. Each yield updates the UI in real time.

Anti-Pattern #3
🔴 Symptom

App grows to 10+ features and becomes impossible to maintain. New functions break old ones.

🟡 Root Cause

Using Gradio beyond its sweet spot. When you have multi-page navigation, session management, database integration, and complex state — Gradio fights you.

🟢 Fix

When your Gradio app needs a sidebar navigation or multiple "pages", migrate to Streamlit. That's the correct tool for that complexity level.

Streamlit

Full Python web apps with sidebar, charts, state, and multi-page navigation
🎬 Origin Story — Why It Was Built

Adrien Treuille and Thiago Teixeira, both with backgrounds at Google and CMU, built Streamlit in 2018 after watching data scientists struggle with the same problem repeatedly: "I have an analysis in Jupyter Notebook. How do I share it as an interactive app without becoming a frontend developer?" The insight was radical: treat the Python script itself as the app. Every time the user interacts, the entire script re-runs top-to-bottom. No callbacks, no event handlers, no state management boilerplate — just write Python and let Streamlit figure out the reactivity. Snowflake acquired Streamlit in 2022 for $800M, validating it as the dominant tool for data app development.

Session State — The Most Critical Concept

Streamlit re-runs the entire script on every interaction. Without st.session_state, your chatbot forgets every message on every reply.

python · production chatbot with streaming + session state
import streamlit as st
from langchain_openai import ChatOpenAI

st.set_page_config(page_title="Chat", page_icon="💬")

# Initialise state ONCE (persists across reruns)
if "messages" not in st.session_state:
  st.session_state.messages = []

# Sidebar
with st.sidebar():
  st.header("⚙️ Settings")
  model = st.selectbox("Model", ["gpt-4o-mini","gpt-4o"])
  temp  = st.slider("Creativity", 0.0, 1.0, 0.7)
  if st.button("🗑️ Clear Chat"):
    st.session_state.messages = []
    st.rerun()

st.title("💬 AI Chat")

# Render history
for msg in st.session_state.messages:
  with st.chat_message(msg["role"]):
    st.markdown(msg["content"])

# New input
if prompt := st.chat_input("Ask me anything..."):
  st.session_state.messages.append({"role":"user","content":prompt})
  with st.chat_message("user"): st.markdown(prompt)

  with st.chat_message("assistant"):
    llm = ChatOpenAI(model=model, temperature=temp, streaming=True)
    # st.write_stream handles the streaming automatically!
    response = st.write_stream(llm.stream(prompt))

  st.session_state.messages.append({"role":"assistant","content":response})

Anti-Patterns — The Mistakes Everyone Makes

Anti-Pattern #1
🔴 Symptom

Chat history disappears after every message. User types a question, gets an answer, types another question — history is gone.

🟡 Root Cause

Storing chat history in a regular Python list. Streamlit reruns the entire script on every interaction, resetting any variable not in session_state.

🟢 Fix

Always initialise: if "messages" not in st.session_state: st.session_state.messages = []. Append to st.session_state.messages, never to a local list.

Anti-Pattern #2
🔴 Symptom

App re-indexes all documents on every page interaction. Users wait 30 seconds for an index that was already built.

🟡 Root Cause

Calling VectorStoreIndex.from_documents() at module level without @st.cache_resource. Streamlit reruns the script — so the index rebuilds every time.

🟢 Fix

Wrap expensive one-time operations: @st.cache_resource for models and indexes, @st.cache_data for data that can be serialised. Runs once, cached forever.

Anti-Pattern #3
🔴 Symptom

Under load, 5 simultaneous users slow each other down. One user's query blocks another's.

🟡 Root Cause

Streamlit uses a single-threaded model by default. Each user session gets one thread; blocking LLM calls on one thread delay all others if using shared resources.

🟢 Fix

Use async LLM calls where possible. For high-concurrency internal tools, configure Streamlit with multiple workers or move the LLM calls into the FastAPI backend which handles async concurrency properly.

Next.js + Vercel AI SDK

Production-grade AI web apps with streaming, voice, and full customization
🎬 Origin Story — Why It Was Built

Next.js was built by Vercel (formerly Zeit) in 2016 as a React framework that adds server-side rendering, file-based routing, and API routes to make production React apps practical. In 2023, as LLMs exploded, Vercel built the Vercel AI SDK on top of Next.js specifically for AI apps — providing React hooks for streaming (useChat, useCompletion), built-in streaming response helpers, and support for OpenAI, Anthropic, and open-source models. The result is the only tool in this module that can handle real-time voice AI via OpenAI's Realtime API — enabling sub-300ms voice conversations in a browser.

❌ Streaming without the AI SDK — painful
// Building streaming from scratch in Next.js
// app/api/chat/route.ts
export async function POST(req: Request) {
  const { message } = await req.json()
  const encoder = new TextEncoder()

  // Manually create a ReadableStream
  const stream = new ReadableStream({
    async start(controller) {
      const response = await openai.chat.completions.create({
        model: 'gpt-4o-mini', stream: true,
        messages: [{ role: 'user', content: message }]
      })
      for await (const chunk of response) {
        const text = chunk.choices[0]?.delta?.content ?? ''
        controller.enqueue(encoder.encode(text))
      }
      controller.close()
    }
  })
  return new Response(stream)
}

✅ With Vercel AI SDK — 5 lines each side
// app/api/chat/route.ts
import { openai } from '@ai-sdk/openai'
import { streamText } from 'ai'

export async function POST(req: Request) {
  const { messages } = await req.json()
  const result = await streamText({
    model: openai('gpt-4o-mini'),
    messages,
  })
  return result.toDataStreamResponse()
}

The Full Streaming Chat UI — Production Pattern

tsx · app/page.tsx — production chatbot with typing indicator + history
'use client'
import { useChat } from 'ai/react'
import { useRef, useEffect } from 'react'

export default function ChatPage() {
  const { messages, input, handleInputChange,
          handleSubmit, isLoading, error } = useChat()
  const bottomRef = useRef<HTMLDivElement>(null)

  // Auto-scroll to latest message
  useEffect(() => {
    bottomRef.current?.scrollIntoView({ behavior: 'smooth' })
  }, [messages])

  return (
    <div className="flex flex-col h-screen max-w-2xl mx-auto p-4">
      <h1 className="text-2xl font-bold mb-4">🤖 AI Chat</h1>

      {/* Message list */}
      <div className="flex-1 overflow-y-auto space-y-3 mb-4">
        {messages.map(m => (
          <div key={m.id}
            className={`p-3 rounded-lg max-w-prose ${
              m.role === 'user'
                ? 'bg-blue-100 ml-auto text-right'
                : 'bg-gray-100'
            }`}>
            <span className="font-semibold text-sm opacity-60">
              {m.role === 'user' ? 'You' : 'AI'}
            </span>
            <p className="mt-1">{m.content}</p>
          </div>
        ))}
        {isLoading && (
          <div className="text-gray-400 animate-pulse p-3">
            Thinking...
          </div>
        )}
        <div ref={bottomRef} />
      </div>

      {/* Input form */}
      <form onSubmit={handleSubmit} className="flex gap-2">
        <input
          value={input}
          onChange={handleInputChange}
          placeholder="Type a message..."
          disabled={isLoading}
          className="flex-1 border rounded-lg px-4 py-2 focus:ring-2"
        />
        <button type="submit" disabled={isLoading}
          className="bg-blue-500 text-white px-6 py-2 rounded-lg">
          {isLoading ? '...' : 'Send'}
        </button>
      </form>
    </div>
  )
}

Anti-Patterns — The Mistakes Everyone Makes

Anti-Pattern #1
🔴 Symptom

OpenAI API key exposed in browser network tab. Security team shuts the app down in hours.

🟡 Root Cause

Calling the OpenAI API directly from a React component (client-side). All API requests are visible in the browser's Network tab, including headers with the API key.

🟢 Fix

Always route API calls through a Next.js API Route (app/api/). The API key stays on the server. The browser only ever hits your own domain. Never use NEXT_PUBLIC_OPENAI_KEY.

Anti-Pattern #2
🔴 Symptom

Chat history disappears on page refresh. Users lose context when they navigate away and come back.

🟡 Root Cause

Using only React state (useState) for messages. React state lives in memory — a page refresh resets it to the initial value.

🟢 Fix

Persist messages to a database (Vercel Postgres, PlanetScale, or Supabase) linked to a session ID. Load history on component mount. The AI SDK's initialMessages prop accepts pre-loaded history.

Anti-Pattern #3
🔴 Symptom

Streaming works perfectly in development but shows nothing until the entire response completes in production (Vercel deployment).

🟡 Root Cause

Vercel's default Serverless Functions buffer responses. The stream isn't flushed to the client until the function completes.

🟢 Fix

Use Edge Runtime for streaming routes: add export const runtime = 'edge' at the top of your API route file. Edge functions support streaming natively without buffering.

import streamlit as st, requests

API = "http://localhost:8000"  # your Module 2 FastAPI
st.set_page_config(page_title="📚 Doc Assistant", layout="wide")

# Sidebar: upload + index
with st.sidebar():
  st.header("📂 Documents")
  files = st.file_uploader("Upload PDF or TXT",
                             type=["pdf","txt"], accept_multiple_files=True)
  if files and st.button("📥 Index"):
    for f in files:
      r = requests.post(f"{API}/upload",
            files={"file": (f.name, f.getvalue())})
      if r.status_code == 200:
        st.success(f"✅ {f.name}: {r.json()['chunks_indexed']} chunks")
  st.divider()
  if st.button("🗑️ Clear Chat"):
    st.session_state.messages = []; st.rerun()

st.title("📚 Document AI Assistant")

if "messages" not in st.session_state:
  st.session_state.messages = []

for m in st.session_state.messages:
  with st.chat_message(m["role"]): st.markdown(m["content"])
  if m.get("sources"):
    with st.expander("📎 Sources"): st.caption(m["sources"])

if q := st.chat_input("Ask about your documents..."):
  st.session_state.messages.append({"role":"user","content":q})
  with st.chat_message("user"): st.markdown(q)

  with st.chat_message("assistant"):
    box     = st.empty()
    partial = ""
    sources = ""
    with requests.post(f"{API}/chat",
        json={"question":q}, stream=True) as resp:
      for chunk in resp.iter_content(chunk_size=None, decode_unicode=True):
        if "__SOURCES__:" in chunk:
          sources = chunk.replace("__SOURCES__:","").strip()
        else:
          partial += chunk; box.markdown(partial + "▌")
    box.markdown(partial)
    if sources:
      with st.expander("📎 Sources"): st.caption(sources)
  st.session_state.messages.append({
    "role":"assistant","content":partial,"sources":sources})
